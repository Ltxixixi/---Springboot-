## 二、推荐算法详细设计

### 2.1 协同过滤模块 (SpotCollaborativeFilteringService)

#### 2.1.1 用户-景点交互矩阵

```sql
-- 交互得分权重
VIEW_WEIGHT = 1.0      -- 浏览
FAVORITE_WEIGHT = 5.0   -- 收藏
SCORE_WEIGHT = 4.0      -- 评分 (满分5分制)
ORDER_WEIGHT = 8.0      -- 购票
```

**矩阵构建**：
```java
// 用户u对景点i的交互得分 = Σ(交互类型权重 × 归一化评分)
interactionScore = (rating - 3) × SCORE_WEIGHT  // 评分归一化
```

#### 2.1.2 ItemCF算法

**核心思想**：用户喜欢景点A，景点A与景点B相似，则推荐景点B

**相似度计算（余弦相似度）**：
```
sim(i,j) = Σ(u_i × u_j) / (||u_i|| × ||u_j||)

其中：
- u_i: 用户对景点i的评分向量
- u_j: 用户对景点j的评分向量
```

**推荐得分计算**：
```java
// 用户u对景点j的推荐得分
score(u,j) = Σ[ sim(i,j) × rating(u,i) ]

其中：
- i: 用户u已交互的景点
- sim(i,j): 景点i与景点j的相似度
- rating(u,i): 用户u对景点i的评分
```

#### 2.1.3 关键代码

```java
// 1. 构建景点相似度矩阵
public Map<Long, List<SpotSimilarity>> buildSpotSimilarityMatrix() {
    // 计算所有景点两两之间的相似度
    for (spot i : allSpots) {
        for (spot j : allSpots) {
            double sim = cosineSimilarity(getRatingVector(i), getRatingVector(j));
            if (sim > threshold) {
                similarityMatrix[i].add(new SpotSimilarity(j, sim));
            }
        }
    }
    return similarityMatrix;
}

// 2. ItemCF推荐
public Map<Long, Double> itemCfRecommend(Long userId, Set<Long> candidates, int topN) {
    for (spot in userInteractions) {
        for (similar in similarityMatrix[spot]) {
            score[similar.spotId] += similar.sim × userRating;
        }
    }
    return topN(score);
}
```

### 2.2 排序模块 (SpotRecommendService)

#### 2.2.1 多特征加权排序

```java
Score = w₁·CF + w₂·TagMatch + w₃·HotScore + w₄·BudgetFit + w₅·CrowdFit

其中：
- CF:        协同过滤得分 (归一化到[0,1])
- TagMatch:  标签匹配得分 (0.3~1.0)
- HotScore:  热度得分 = view×0.4 + favour×0.3 + avgScore×0.3
- BudgetFit: 预算适配得分 (0.2~1.0)
- CrowdFit:  人群适配得分 (0.3~1.0)
```

#### 2.2.2 权重配置

| 特征 | 权重 | 说明 |
|------|------|------|
| CF | 0.25 | 协同过滤得分 |
| TagMatch | 0.25 | 标签匹配度 |
| HotScore | 0.25 | 热度指数 |
| BudgetFit | 0.15 | 预算适配 |
| CrowdFit | 0.10 | 人群适配 |

#### 2.2.3 标签匹配得分计算

```java
// 余弦相似度计算标签匹配
double tagScore = cosineSimilarity(userTagVector, spotTagVector);

// 或加权标签匹配
for (tag in spotTags) {
    totalScore += userPreference.getOrDefault(tag, 0);
}
tagScore = totalScore / spotTags.size();
```

#### 2.3.2 MMR重排序

```java
public List<Spot> mmrRerank(List<Spot> candidates, int topK) {
    List<Spot> result = new ArrayList<>();
    Set<String> selectedTags = new HashSet<>();

    for (int i = 0; i < topK && !candidates.isEmpty(); i++) {
        // 计算每个候选的MMR分数
        for (Spot candidate : candidates) {
            double relevance = candidate.getScore();
            double diversity = calculateDiversity(candidate, result);
            double mmrScore = λ × relevance - (1-λ) × diversity;

            candidate.setMmrScore(mmrScore);
        }

        // 选择MMR分数最高的
        Spot selected = selectMaxMmr(candidates);
        result.add(selected);
        selectedTags.addAll(selected.getTags());
    }

    return result;
}
```

### 3.2 日内排序

#### 3.2.1 时段优化

```java
// 时段优先级
BLOCK_MORNING = 1;    // 上午: 山岳、湖泊、寺庙
BLOCK_AFTERNOON = 2;  // 下午: 博物馆、纪念馆、室内

// 日内排序: 先按时段，再按热度
spots.sort((a, b) -> {
    if (a.timeBlock != b.timeBlock) {
        return a.timeBlock - b.timeBlock;
    }
    return b.score - a.score;
});
```

#### 3.2.2 地理最近邻

```java
// 贪心最近邻排序
public List<Spot> greedyNearestNeighbor(List<Spot> spots) {
    List<Spot> route = new ArrayList<>();
    List<Spot> remaining = new ArrayList<>(spots);
    Spot current = remaining.remove(0);

    while (!remaining.isEmpty()) {
        Spot nearest = findNearest(current, remaining);
        route.add(nearest);
        remaining.remove(nearest);
        current = nearest;
    }

    return route;
}
```

### 3.3 遗传算法优化

#### 3.3.1 染色体编码

```java
// 染色体: 景点ID序列
chromosome = [3, 1, 5, 2, 4]  // 表示路线: 景点3 → 景点1 → 景点5 → 景点2 → 景点4
```

#### 3.3.2 适应度函数

```java
// 适应度 = 1 / 总成本
// 总成本 = 距离成本 + 时间惩罚 + 体验惩罚

Fitness = 1 / (Distance × W₁ + TimePenalty × W₂ + ExperiencePenalty × W₃)

// 其中：
// - Distance: 景点间总距离 (km)
// - TimePenalty: 超出游玩时间的惩罚
// - ExperiencePenalty: 不合理游玩顺序的惩罚 (如连续游览同类型景点)
```

#### 3.3.3 遗传算子

| 算子 | 实现 | 说明 |
|------|------|------|
| 选择 | 锦标赛选择 | 每次从种群中选3个，返回最优 |
| 交叉 | 顺序交叉(OX) | 保留父代1的片段顺序 |
| 变异 | 逆序变异+插入变异 | 50%概率逆序，50%概率插入 |

#### 3.3.4 算法参数

```java
DEFAULT_POPULATION_SIZE = 50;   // 种群大小
DEFAULT_MAX_ITERATIONS = 100;  // 最大迭代次数
DEFAULT_MUTATION_RATE = 0.15;  // 变异率
ELITE_COUNT = 5;               // 精英保留数
```

#### 3.3.5 关键代码

```java
public GeneticRouteResult optimizeDailyRoute(List<Spot> spots, ...) {
    // 1. 初始化种群
    List<Long[]> population = initializePopulation(spots, POPULATION_SIZE);

    // 2. 遗传迭代
    for (int iter = 0; iter < maxIterations; iter++) {
        // 评估适应度
        List<Double> fitness = evaluate(population, spotMap);

        // 选择
        List<Long[]> selected = tournamentSelect(population, fitness);

        // 交叉
        List<Long[]> crossed = orderCrossover(selected);

        // 变异
        List<Long[]> mutated = mutate(crossed, mutationRate);

        // 更新种群
        population = mutated;
    }

    // 3. 返回最优解
    return bestSolution;
}
```

```java
public String chat(String systemPrompt, String userPrompt) {
    // 1. 熔断检查
    if (isCircuitOpen()) {
        return getDegradedResponse();
    }

    // 2. 限流检查
    if (!tryAcquire()) {
        return getDegradedResponse();
    }

    // 3. 并发控制
    semaphore.acquire();

    try {
        // 4. 带重试的调用
        return callWithRetry(systemPrompt, userPrompt);
    } catch (Exception e) {
        // 5. 失败，触发熔断
        onFailure(e);
        throw e;
    } finally {
        semaphore.release();
    }
}
```

## 七、论文表述建议

### 7.1 推荐模块

**标题**：
基于协同过滤与内容特征融合的混合景点推荐算法

**核心表述**：
> 本文采用基于内容与用户行为融合的混合推荐算法。首先，通过分析用户的收藏、评分、购票等行为数据，构建用户-景点交互矩阵，并利用ItemCF协同过滤算法计算景点间的相似度，实现候选景点召回。其次，融合协同过滤得分、标签匹配度、热度指数、预算适配度等特征进行加权排序，最后通过MMR多样性重排保证推荐结果的新颖性与多样性。

### 7.2 路线规划模块

**标题**：
基于多约束优化的旅游路线生成算法

**核心表述**：
> 本文提出一种基于多约束优化的旅游路线生成方法。该方法首先根据用户偏好约束过滤候选景点，然后采用贪心策略与装箱算法进行分天规划，在日内采用最近邻排序优化游玩顺序。为进一步提升路线质量，引入了基于遗传算法的局部优化模块，通过设计合理的染色体编码、适应度函数和遗传算子，在可接受的时间复杂度内搜索更优的路线顺序。

### 7.3 AI对话模块

**标题**：
基于大语言模型的用户需求解析与结果解释方法

**核心表述**：
> 本文利用大语言模型强大的自然语言理解能力，将用户的模糊旅游需求解析为结构化偏好向量，并基于提示词工程生成可解释的推荐理由与路线说明。同时，为保障系统稳定性与成本可控，引入了超时控制、并发限流与熔断降级策略。
